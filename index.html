Caso ideal: Si encontramos una expresin que evala exactamente al valor objetivo, entonces f(x) = 0, que es el mnimo absoluto posible.
Casos subptimos: Cualquier expresin que no evale exactamente al valor objetivo tendr f(x) > 0, y mientras mayor sea esta diferencia, peor ser la solucin.

Definicin de la Funcin Objetivo

El problema consiste en encontrar expresiones matemticas que evalen a un valor objetivo especfico. La funcin objetivo se define como:

$$f(x) = |resultado_{expresin} - valor_{objetivo}|$$

Donde:

  • $resultado_{expresin}$ = valor numrico al evaluar la expresin matemtica
  • $valor_{objetivo}$ = nmero que queremos obtener (ej: 4)
  • $|...|$ = valor absoluto de la diferencia

Justificacin: Es un Problema de Minimizacin

Este es un problema de minimizacin porque:

  • Objetivo: Minimizar la diferencia entre el resultado y el valor deseado
  • ptimo global: $f(x) = 0$ (cuando encontramos el valor exacto)
  • Direccin de mejora: Valores menores de $f(x)$ son mejores

Casos de Anlisis

Caso Descripcin Valor de f(x) Calidad
ptimo $resultado = objetivo$ $f(x) = 0$ Mejor posible
Subptimo cercano $resultado objetivo$ $f(x) > 0$ (pequeo) Buena aproximacin
Subptimo lejano $resultado objetivo$ $f(x) >> 0$ (grande) Mala solucin

def permutaciones(lista, n):
    if n == 0:
        return [[]]
    resultado = []
    for i in range(len(lista)):
        elem = lista[i]
        resto = lista[:i] + lista[i+1:]
        for p in permutaciones(resto, n-1):
            resultado.append([elem] + p)
    return resultado

from itertools import permutations
import time

def fuerza_bruta_optimizada(valor_objetivo=4):
    """Referencia: fuerza bruta con itertools"""
    cifras = [1, 2, 3, 4, 5, 6, 7, 8, 9]
    operadores = ['+', '-', '*', '/']
    
    soluciones = []
    evaluaciones = 0
    
    for perm_cifras in permutations(cifras, 5):
        for perm_ops in permutations(operadores, 4):
            expr_str = f"{perm_cifras[0]}{perm_ops[0]}{perm_cifras[1]}{perm_ops[1]}{perm_cifras[2]}{perm_ops[2]}{perm_cifras[3]}{perm_ops[3]}{perm_cifras[4]}"
            
            try:
                resultado = eval(expr_str)
                evaluaciones += 1
                
                if abs(resultado - valor_objetivo) < 1e-10:
                    soluciones.append((expr_str, resultado))
            except ZeroDivisionError:
                evaluaciones += 1
                continue
    
    return soluciones, evaluaciones

print("=== FUERZA BRUTA OPTIMIZADA (REFERENCIA) ===")
start = time.time()
sol_ref, eval_ref = fuerza_bruta_optimizada(4)

Ahora vamos a implementar otra alternativa usando un branch and bound, la idea es la siguiente:

  1. Dividir el problema en subproblemas ms pequeos (como las ramas de un rbol). Cada nodo representa una posible solucin parcial.

  2. Para cada subproblema, se calcula una cota superior o inferior del mejor resultado que puede lograrse desde ese nodo.

  3. Se evitan explorar ramas del rbol que no pueden mejorar la solucin ptima actual, lo que ahorra tiempo y recursos.

def branch_and_bound_correcto(valor_objetivo=4):
    """Branch and Bound que replica la lgica exacta pero con podas por cotas"""
    cifras = [1, 2, 3, 4, 5, 6, 7, 8, 9]
    operadores = ['+', '-', '*', '/']
    
    soluciones = []
    evaluaciones = 0
    podas = 0
    
    # Usar la misma estructura que fuerza bruta pero con podas
    for perm_cifras in permutations(cifras, 5):
        # Poda rpida por cotas de nmeros solamente
        suma_total = sum(perm_cifras)
        diferencia_maxima = perm_cifras[0] - sum(perm_cifras[1:])
        
        # Si ni la suma total ni la diferencia mxima se acercan al objetivo
        if suma_total < valor_objetivo - 2 or diferencia_maxima > valor_objetivo + 2:
            podas += 1
            continue
        
        for perm_ops in permutations(operadores, 4):
            # Evaluacin idntica a fuerza bruta
            expr_str = f"{perm_cifras[0]}{perm_ops[0]}{perm_cifras[1]}{perm_ops[1]}{perm_cifras[2]}{perm_ops[2]}{perm_cifras[3]}{perm_ops[3]}{perm_cifras[4]}"
            
            try:
                resultado = eval(expr_str)
                evaluaciones += 1
                
                if abs(resultado - valor_objetivo) < 1e-10:
                    soluciones.append((expr_str, resultado))
            except ZeroDivisionError:
                evaluaciones += 1
                continue
    
    return soluciones, evaluaciones, podas

Ventajas: Estrategia de Poda por Cotas

Esta implementacin aade una estrategia de poda por cotas al algoritmo de fuerza bruta, evitando explorar combinaciones de cifras que matemticamente no pueden alcanzar el valor objetivo.

Criterio de poda:
Si estas cotas estn muy lejos del objetivo, se descartan todas las combinaciones de operadores para esa permutacin de cifras.

Paralelizacin con Multiprocessing

Qu se mejor?

Esta implementacin distribuye el trabajo entre mltiples procesos (cores) del CPU, permitiendo evaluar expresiones en paralelo sin cambiar la lgica del algoritmo.

Estrategia de Paralelizacin

  • Divisin del trabajo: Las 15,120 permutaciones de cifras se dividen equitativamente entre los procesos disponibles
  • Procesamiento independiente: Cada proceso evala su bloque de permutaciones todas las permutaciones de operadores
  • Combinacin de resultados: Al final se unen todas las soluciones encontradas por cada proceso
